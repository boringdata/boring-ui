## Boring-ui Customizations

Repo-owned customizations layered on top of shadcn baseline:

- `src/front/styles.css` remains the host-owned shared stylesheet entry
- `src/front/styles/tokens.css` remains the source of boring-ui token and theme bridge
- `src/front/lib/utils.js` (`cn`) is the repo-owned class-composition utility contract
- generated primitive imports were normalized from `@/lib/utils` to relative imports
  so Vitest/build resolution remains deterministic in this repo's current config
- root package CSS/runtime contracts are documented in:
  - `docs/runbooks/CSS_CONTRACT.md`
  - `docs/runbooks/SHARED_STYLE_RUNTIME_CONTRACT.md`

## Update Rule

If a file originally generated by shadcn is manually edited:

1. keep the path under "Generated From shadcn"
2. add a short note here explaining what was customized and why
3. preserve deterministic regeneration instructions in `SHADCN_BASELINE.md`
